# ADR-041 — EmailProtocol Interface

**Status:** Accepted  
**Phase:** 6  
**Resolves:** I1 known gap (email backend has no Protocol); enables email verification & password reset routes

## Context

ADR-018 established `aiosmtplib` as the email delivery library and allows custom backends via `email_backend` dotted-path setting. However, no `EmailProtocol` was defined — the only backend family without a formal interface. INVARIANTS.md I1 explicitly notes this as a known gap. Phase 6 needs to implement `POST /auth/send-verification`, `POST /auth/verify-email`, `POST /auth/forgot-password`, and `POST /auth/reset-password`, all of which require sending email.

Design constraints:
- Must be async (I2)
- Must support the verification/reset use case (HTML + plain text, subject, recipient)
- Must NOT over-engineer — transactional email is the only use case (no attachments, no CC/BCC at protocol level)
- Must allow custom backends via ADR-012 dotted-path

## Decision

### EmailProtocol

```python
from typing import Protocol

class EmailProtocol(Protocol):
    async def send(
        self,
        *,
        to: str,
        subject: str,
        body_text: str,
        body_html: str | None = None,
    ) -> None:
        """Send a single transactional email.

        Args:
            to: Recipient email address
            subject: Email subject line
            body_text: Plain text body (required; used as fallback)
            body_html: Optional HTML body (multipart/alternative when provided)

        Raises:
            EmailDeliveryError: If sending fails after retries (if any)
        """
        ...
```

### SmtpEmailBackend (built-in)

```python
class SmtpEmailBackend:
    def __init__(self, settings) -> None:
        try:
            import aiosmtplib
        except ImportError:
            raise ImportError(
                "SMTP email backend requires aiosmtplib. "
                "Install it with: pip install fast-agent-stack[email-smtp]"
            )
        self._host = settings.smtp_host
        self._port = settings.smtp_port
        self._username = settings.smtp_username
        self._password = settings.smtp_password
        self._use_tls = settings.smtp_use_tls
        self._from_addr = settings.email_from
        self._from_name = settings.email_from_name

    async def send(self, *, to, subject, body_text, body_html=None) -> None:
        ...
```

### Factory

```python
def get_email_backend(settings) -> EmailProtocol:
    backend = settings.email_backend
    match backend:
        case "smtp":
            return SmtpEmailBackend(settings)
        case _ if "." in backend:
            # ADR-012 dotted-path custom backend
            ...
        case _:
            raise ValueError(f"Unknown email backend: {backend!r}")
```

### Exception

`EmailDeliveryError(Exception)` — defined in `core/email/__init__.py`. Raised when delivery fails. Auth routes catch this, log it, and return success to the user (fire-and-forget semantics per ADR-018 — failures don't abort the HTTP response).

### Module Location

```
core/email/
    __init__.py     — EmailProtocol, EmailDeliveryError, get_email_backend
    smtp.py         — SmtpEmailBackend
```

Not under `core/auth/` because email is a general-purpose service — future phases may send non-auth emails (notifications, alerts).

### Settings (already defined in ADR-018, confirmed here)

```python
email_backend: str = "smtp"
smtp_host: str = "localhost"
smtp_port: int = 587
smtp_username: str | None = None
smtp_password: str | None = None
smtp_use_tls: bool = True
email_from: str = "noreply@example.com"
email_from_name: str = "FastAgentStack"
```

## Consequences

- I1 known gap is resolved — `EmailProtocol` is now defined
- I1 `Applies to` section should be amended to include `core/email/`
- Custom email backends (SES, SendGrid) follow ADR-012: `email_backend = "myproject.email.SESBackend"`
- Auth verification routes use `get_email_backend(settings)` to send tokens
- Fire-and-forget: `send()` is awaited but delivery failures are logged and swallowed at the route level — the user gets a 200 regardless (prevents email enumeration attacks)
- No batch sending at protocol level — transactional emails are always single-recipient
- No attachment support at protocol level — out of scope for auth flows; custom backends can extend

## Rejected Alternatives

- **No Protocol (inline aiosmtplib calls)** — rejected: violates ADR-012 custom backend contract; makes testing impossible without SMTP server
- **`send_bulk()` method** — deferred: not needed for auth flows; can be added later without breaking the protocol
- **Attachments in Protocol** — rejected: over-engineering for transactional auth emails; custom backends can support it independently
- **Return delivery ID** — rejected: SMTP doesn't reliably return one; would force all backends to fabricate IDs
