# ADR-050: Per-Backend Base URL and Model Settings

**Status:** Accepted
**Date:** 2026-07-13
**Deciders:** Author

## Context

The `openai` backend for both LLM and embedding uses the `AsyncOpenAI` client, which reads
`OPENAI_API_KEY` and `OPENAI_BASE_URL` from environment by default. This is problematic:

1. A user may want different providers for LLM vs embedding (e.g., OpenAI for chat, Ollama for
   embedding, or Bedrock for embedding and Anthropic for chat).
2. The `OPENAI_BASE_URL` env var is not namespaced with the project prefix, breaking the
   framework's `env_prefix` convention.
3. Ollama exposes an OpenAI-compatible API at `{ollama_url}/v1`, but there is no framework
   setting to configure this per-backend.

## Decision

Each backend family that uses an HTTP client gets its own `base_url` and `model` setting in
`BaseSettings`:

```python
# LLM (OpenAI-compatible backend)
llm_base_url: str | None = None
llm_model: str = "gpt-4o"

# Embedding (OpenAI-compatible backend)
embedding_base_url: str | None = None
embedding_model: str = "text-embedding-3-small"

# Reranker
reranker_url: str = "http://localhost:11434"  # already exists
reranker_model: str = "qllama/bge-reranker-v2-m3"  # already exists
```

When `base_url` is `None`, the backend uses its SDK default (e.g., `https://api.openai.com/v1`
for OpenAI). When set, it overrides to point at Ollama, Azure OpenAI, or any compatible endpoint.

### Backend wiring

```python
# OpenAI LLM backend
self._client = AsyncOpenAI(
    base_url=settings.llm_base_url,
    timeout=settings.llm_timeout,
)

# OpenAI Embedding backend
self._client = AsyncOpenAI(
    base_url=settings.embedding_base_url,
    timeout=settings.embedding_timeout,
)
```

### Tutorial .env (Ollama for both)

```bash
DOCQA_LLM_BASE_URL=http://localhost:11434/v1
DOCQA_LLM_MODEL=llama3.2
DOCQA_EMBEDDING_BASE_URL=http://localhost:11434/v1
DOCQA_EMBEDDING_MODEL=nomic-embed-text
```

### Mix-and-match example (OpenAI chat + Ollama embedding)

```bash
DOCQA_LLM_BASE_URL=              # empty = default OpenAI endpoint
DOCQA_LLM_MODEL=gpt-4o
DOCQA_EMBEDDING_BASE_URL=http://localhost:11434/v1
DOCQA_EMBEDDING_MODEL=nomic-embed-text
```

## Naming convention

The pattern is `{family}_{setting}`:

| Family | Settings |
|--------|----------|
| `llm_` | `llm_base_url`, `llm_model`, `llm_timeout` |
| `embedding_` | `embedding_base_url`, `embedding_model`, `embedding_timeout` |
| `reranker_` | `reranker_url`, `reranker_model`, `reranker_timeout` |
| `vector_` | `vector_url`, `vector_timeout` |
| `storage_` | `storage_timeout` |

This supersedes `embedding_openai_model` (renamed to `embedding_model` for provider-neutrality).

## Consequences

| File | Change |
|------|--------|
| `core/config.py` | Add `llm_base_url`, `llm_model`, `embedding_base_url`, `embedding_model` |
| `core/ai/llm/openai.py` | Read `settings.llm_base_url` and `settings.llm_model` |
| `core/ai/embedding/backends/openai.py` | Read `settings.embedding_base_url` and `settings.embedding_model` |
| Template `.env.example.jinja` | Add `LLM_BASE_URL`, `LLM_MODEL`, `EMBEDDING_BASE_URL`, `EMBEDDING_MODEL` |
| Template `settings.py.jinja` | Add new fields |
| `docs/tutorial/00-prerequisites.md` | Update env var list |
| `docs/tutorial/04-ingestion-agent.md` | Reference correct settings |

### Behavioral consequences

- Each backend is independently configurable via env vars
- Ollama works by setting `*_BASE_URL=http://localhost:11434/v1` per backend
- No implicit coupling between LLM and embedding providers
- `OPENAI_API_KEY` still works (AsyncOpenAI reads it from env) but `base_url` is controlled
  by the framework setting, not `OPENAI_BASE_URL`
- Rename: `embedding_openai_model` becomes `embedding_model` (breaking for existing configs)
