# ADR-047: CLI Scaffolder Interactive Question Sequence

**Status:** Accepted
**Date:** 2026-07-12
**Deciders:** Author

## Context

`fas new` prompts users for project configuration interactively. The original implementation asked
questions in the order they appeared in `copier.yml`, which felt like a form rather than a guided
setup. Users reported the sequence felt unnatural.

The framework serves two audiences: traditional web API developers (like Django) and AI agent
builders. The question sequence must serve both without assuming AI is always the goal.

## Decision

Questions follow a "secure it, store it, make it smart, show it, observe it, ship it" mental model:

```
1.  Database
2.  Authentication -> Auth backend -> Email
3.  SQLAdmin
4.  Rate limiting
5.  File storage
6.  Task broker -> Scheduler
7.  LLM provider -> Vector DB -> Embedding provider
8.  Chat frontend
9.  Tracing
10. Secrets manager -> Dockerfile -> docker-compose -> Kubernetes
```

### Grouping rationale

| Group | Questions | Rationale |
|-------|-----------|-----------|
| Security layer | DB, Auth, Admin, Rate limit | Who can access, how they're managed, how they're protected |
| Data/infra | Storage, Tasks | What you persist and how you process it asynchronously |
| AI | LLM, Vector DB, Embedding | Builds on the data layer |
| UI | Chat frontend | Only meaningful with an LLM wired up |
| Ops | Tracing | Production observability |
| Deployment | Secrets, Dockerfile, docker-compose, K8s | Ship it |

### Conditional questions

Some questions only appear when a prior answer makes them relevant:

| Condition | Questions shown |
|-----------|-----------------|
| `include_auth == true` | Auth backend, Email |
| `task_broker != "none"` | Scheduler |
| `llm_provider != "none"` | Vector DB, Embedding provider |
| `vector_db != "none"` | Embedding provider |
| `llm_provider != "none"` | Chat frontend |
| `include_dockerfile == true` | docker-compose, Kubernetes |

### Presets (deferred)

Presets will define which questions to surface, silently filling the rest. This is deferred until all
flags are implemented and we can determine which questions each preset should ask. Until then, the
default flow is "custom" (all questions asked).

### CLI flag overrides

Any question can be skipped by passing its corresponding CLI flag:

```bash
fas new myapp --db postgres --llm openai --no-auth
```

Flags set the value and suppress the prompt for that question. The `-y` / `--defaults` flag accepts
all defaults without prompting (requires `--preset`).

## Consequences

- Question sequence is fixed regardless of `copier.yml` ordering
- All prompting logic lives in `cli/new.py`, not in copier's interactive mode
- copier always runs with `defaults=True` since data is fully gathered before invocation
- Adding a new copier variable requires placing it in the correct position in the sequence
- Preset behavior is deferred; the scaffolder always asks all relevant questions for now
