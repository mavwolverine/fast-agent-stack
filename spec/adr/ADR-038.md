# ADR-038 — Phase 5 Protocol Signatures: StorageProtocol, VectorStoreProtocol, EmbeddingProtocol

**Status:** Accepted  
**Phase:** 5  
**Resolves:** ND-1 (Protocol method signatures undefined)

## Context

ARCHITECTURE.md lists method names for the three Phase 5 backend families but provides no parameter types or return types. Without full signatures, I1 (Full Protocol Conformance) cannot be verified across four implementations per family. ADR-036 established this pattern for the LLM family; this ADR does the same for storage, vector, and embedding.

Design constraints:
- All I/O must be async (I2)
- Backends receive `Settings` at construction (ADR-012 custom backend contract)
- Return types must be simple, serializable, and framework-owned (no numpy/pandas in the protocol)
- Vector store must accept pre-computed vectors (separation of concerns: embedding is a separate step)

## Decision

### StorageProtocol

```python
from typing import Protocol

class StorageProtocol(Protocol):
    async def upload(
        self,
        key: str,
        data: bytes,
        *,
        content_type: str = "application/octet-stream",
    ) -> str:
        """Upload data. Returns the canonical key (may include prefix/normalization)."""
        ...

    async def download(self, key: str) -> bytes:
        """Download data by key. Raises KeyNotFoundError if missing."""
        ...

    async def delete(self, key: str) -> None:
        """Delete object by key. No-op if key does not exist."""
        ...

    async def exists(self, key: str) -> bool:
        """Check if key exists in the store."""
        ...

    async def url(self, key: str, *, expires_in: int = 3600) -> str:
        """Generate a URL for the object. For local storage, returns a relative path.
        For remote storage, returns a pre-signed URL valid for `expires_in` seconds."""
        ...
```

**Key design choices:**
- `upload` takes `bytes` not `IO[bytes]` — simpler contract, no seek/tell issues across async boundaries. Callers stream large files via chunked upload at the HTTP layer; the protocol operates on complete byte payloads. A future `upload_stream` method may be added if needed.
- `upload` returns `str` (the canonical key) to support backends that normalize paths.
- `delete` is idempotent — no error on missing key (matches S3 semantics).
- `url` has `expires_in` for pre-signed URLs; local backend returns a static path and ignores the parameter.

**Exception:** `KeyNotFoundError` (defined in `core/storage/__init__.py`, subclasses `Exception`).

### VectorStoreProtocol

```python
from dataclasses import dataclass
from typing import Protocol

@dataclass(frozen=True)
class VectorSearchResult:
    id: str
    score: float
    metadata: dict[str, str | int | float | bool]
    content: str | None  # original text if stored; None if metadata-only

class VectorStoreProtocol(Protocol):
    async def create_collection(
        self,
        name: str,
        dimensions: int,
        *,
        distance_metric: str = "cosine",
    ) -> None:
        """Create a collection/index. No-op if it already exists."""
        ...

    async def upsert(
        self,
        collection: str,
        id: str,
        vector: list[float],
        metadata: dict[str, str | int | float | bool],
        *,
        content: str | None = None,
    ) -> None:
        """Insert or update a single vector with metadata."""
        ...

    async def search(
        self,
        collection: str,
        vector: list[float],
        *,
        top_k: int = 10,
        filter: dict[str, str | int | float | bool] | None = None,
    ) -> list[VectorSearchResult]:
        """Search by pre-computed query vector. Returns top_k results ordered by similarity."""
        ...

    async def delete(self, collection: str, id: str) -> None:
        """Delete a vector by ID. No-op if not found."""
        ...

    async def close(self) -> None:
        """Release underlying connections/resources."""
        ...
```

**Key design choices:**
- `search` takes `vector: list[float]` (pre-computed), not a text query. Embedding is a separate concern handled by `EmbeddingProtocol`. This allows users to choose their embedding strategy independently of the vector store.
- `metadata` is `dict[str, str | int | float | bool]` — the common denominator across all four backends. Complex nested metadata is not supported at the protocol level (Qdrant supports it; pgvector/OpenSearch don't).
- `content` field on upsert and results allows storing the original text alongside the vector for retrieval without a secondary lookup.
- `filter` on search uses the same flat metadata dict shape — backends translate to their native filter syntax.
- `distance_metric` accepts `"cosine"`, `"euclidean"`, `"dot"`. Backends that don't support a given metric raise `ValueError` at `create_collection` time.
- `create_collection` is idempotent — matches Qdrant and Weaviate semantics; pgvector and OpenSearch implementations check-and-skip.

**Exception:** `CollectionNotFoundError` (defined in `core/vector/__init__.py`, subclasses `Exception`). Raised by `search`, `upsert`, and `delete` if the collection does not exist.

### EmbeddingProtocol

```python
from typing import Protocol

class EmbeddingProtocol(Protocol):
    @property
    def dimensions(self) -> int:
        """The dimensionality of vectors produced by this embedding model."""
        ...

    async def embed(self, text: str) -> list[float]:
        """Embed a single text string. Returns a vector of length `dimensions`."""
        ...

    async def embed_batch(self, texts: list[str]) -> list[list[float]]:
        """Embed multiple texts. Returns vectors in the same order as input.
        Implementations should optimize for batching (fewer API calls)."""
        ...
```

**Key design choices:**
- Returns `list[float]`, not `numpy.ndarray`. This avoids a numpy dependency in the protocol and keeps serialization trivial. Backends that use numpy internally convert on output.
- `dimensions` is a property (not async) because it's model-specific and known at construction time. Backends read it from config or model metadata.
- `embed_batch` exists separately from `embed` because remote APIs (OpenAI, Bedrock) have native batch endpoints that are significantly cheaper/faster than N sequential calls.
- No `model_id` property on the protocol — the model is a configuration concern (in Settings), not a runtime query. Contrast with `LLMBackend.model_id` which is needed for usage logging.

### Construction Contract (all three families)

All backends follow the same construction pattern (ADR-012):

```python
class MyBackend:
    def __init__(self, settings: Settings) -> None:
        ...
```

The factory passes the full `Settings` object. Backends extract what they need. No other constructor signatures are supported.

## Consequences

- I1 conformance is now enforceable: every backend must implement every method with exact signatures.
- `VectorSearchResult` is a framework-owned dataclass exported from `core/vector/__init__.py`.
- `KeyNotFoundError` exported from `core/storage/__init__.py`.
- `CollectionNotFoundError` exported from `core/vector/__init__.py`.
- `ARCHITECTURE.md` Protocol methods table must be updated with full signature details (see below).
- All custom backends (ADR-012 dotted-path) must conform to these exact signatures.
- Type stubs will be included for mypy strict compliance (ADR-011).
- Metadata shape (`dict[str, str | int | float | bool]`) is intentionally restrictive — it is the intersection of what all four vector backends support natively. Users needing nested structures must serialize to string.

## Rejected Alternatives

- **`upload(key, data: IO[bytes])`** — rejected: async I/O on stream objects is complex (no standard async file-like protocol); `bytes` is simpler and sufficient for framework-scale files. Large file streaming is handled at the HTTP route level, not the storage protocol.
- **`search(collection, query: str)`** — rejected: couples vector store to embedding. Violates single-responsibility. RAG service composes the two.
- **`embed() -> numpy.ndarray`** — rejected: adds a required dependency to the protocol contract. `list[float]` is universally serializable.
- **Generic metadata (`dict[str, Any]`)** — rejected: backends would silently drop unsupported types at runtime. Restrictive typing catches errors at write time.
- **Batch upsert (`upsert_batch`)** — deferred: can be added later without breaking the protocol. Single-item upsert is the minimum viable contract. RagService can implement batching by calling `upsert` in a loop with `asyncio.gather`.
